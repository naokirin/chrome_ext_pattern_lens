複数キーワード検索で、キーワード間の最大距離を計算する際の倍率を設定します。範囲 = キーワードの総文字数 × 倍率（例: 倍率6の場合、10文字のキーワードは60文字の範囲内でマッチします）

複数キーワード検索で、キーワード間の最小距離を設定します。短いキーワードでも、この最小範囲が適用されます

複数キーワード検索で、キーワード間の最大距離を設定します。長いキーワードでも、この最大範囲を超えることはありません

機能の排他制御について

一部の検索機能は同時に使用できません。以下の排他制御が適用されます：

  • 要素検索が有効な場合: あいまい検索、大文字小文字の区別、正規表現検索が自動的に無効になります
  • あいまい検索が有効な場合（要素検索が無効な場合）: 大文字小文字の区別と正規表現検索が自動的に無効になります

これらの設定を変更すると、排他制御に基づいて他の設定が自動的に調整されます。

あいまい検索の仕様

  • 複数キーワード検索: 空白で区切られた複数のキーワードが一定範囲内に存在する場合にマッチします
  • ひらがな・カタカナの相互一致: ひらがなとカタカナを区別せずに検索します（例: あいう と アイウ を同一視）
  • アルファベットの大文字小文字の相互一致: アルファベットの大文字と小文字を区別せずに検索します（例: ABC と abc を同一視）
  • 記号の表記ゆれ吸収: ハイフン、伸ばし棒などの半角・全角の差異を吸収します（例: -、ー、－ を同一視）
  • 文字種の表記ゆれ吸収: アルファベット、数字、カタカナの全角・半角の差異を吸収します（例: ABC と ＡＢＣ を同一視）
  • 濁点・半濁点の結合文字正規化: 別文字の濁点（゛）や半濁点（゜）がある場合に、結合した1文字と同じ判定にします（例: か゛、ｶﾞ を が として扱う）
  • アクセント付き文字の正規化: 英語以外のアルファベットの特殊文字を英語のアルファベットに正規化します（例: ç → c、é → e、ñ → n）
  • 代用表記の対応: ドイツ語などの代用表記に対応します（例: ß → ss、ä → ae、ö → oe、ü → ue、œ → oe）
  • 上付き・下付き・丸数字の正規化: 上付き数字（¹、²、³、⁰-⁹）、下付き数字（₀-₉）、丸数字（①-⑳、⓫-⓴、⓵-⓿）を通常の数字として扱います（例: ¹、₁、① を 1 として扱う）
注意: 要素検索モードがONの場合は、あいまい検索は自動的にOFFになります
